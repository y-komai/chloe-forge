2026.04 / エンジニア勉強会 / y-komai

きっかけ：OpenClawとの出会い


「Devin に指示するのすらめんどいよね？
その指示を OpenClaw にやらせようっていう発想。
いわゆる A2A（Agent to Agents）。」

これを聞いたとき、衝撃だった。
AIが 「答える道具」 じゃなくて 「動かす存在」 として日常に溶け込んでいる。
しかも AIがAIに指示する。

OpenClaw って何をしていたのか

  • 常駐 Discord にずっといる。話しかけると反応する
  • 記憶 前の会話を覚えてる。日報を自動でまとめる
  • ツール実行 GitHub を操作する、ファイルを書く、テストを回す
  • AI 連携 他の AI（Devin 等）に指示を出すオーケストレーター

OpenClaw は 独自フレームワーク として、これらすべてを自前で持っていた

ここで気づいたこと


重要なのは
「どう指示するか」じゃなく
「どんな環境で動かすか」だ


ツールは何を使えるか、どこで止まるか、
失敗したらどうするか、何を記憶するか——
この「足場」の設計が結果を決める

OpenClaw はどうなったか


Anthropic の OAuth ポリシー変更で 使用不可 に。
同じ時期に Claude Code という公式プロダクトが登場した。


OpenClaw 時代 現在（Claude Code）
ランタイム自前フレームワークAnthropic 公式プロダクト
オーケストレーターくろえ自身が担うClaude Code 側が担う
ハーネスの実体openclaw の設定・コードCLAUDE.md + memory/ ファイル群
維持コスト自分でフレームワークを育てるAnthropic が育てる

「オーケストレーター」の役割がプラットフォーム側に移管された

業界も同じ方向へ

  • 2024年末 MCP（Model Context Protocol）— Anthropic。AIとツールをつなぐ標準規格
  • 2025年4月 A2A プロトコル — Google → Linux Foundation。AIとAIが通信する標準規格
  • 2026年4月 Claude Managed Agents — Anthropic。オーケストレーション層ごとホストするサービス

「自前で作る必要があったもの」が
標準化・サービス化されていく流れ

まとめ


  • 変わったこと AIは「答える道具」から「自律で動くエージェント」へ
  • 重要になること ハーネスの設計：AIが動く環境・制約・記憶を作る技術
  • 業界の方向 A2A、Claude Managed Agents——「個人レベルでやってたこと」が産業化される

「AIに指示する」の次は
「AIが正しく動く環境を作る」
