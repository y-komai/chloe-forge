コード例 — tool 定義

tools = [{
    "name": "run_ssh",
    "description": "Run a read-only shell command on the target EC2 host. "
                   "Do NOT use destructive commands (rm, kill, shutdown).",
    "input_schema": {
        "type": "object",
        "properties": {
            "host": {
                "type": "string",
                "enum": ["web-01", "web-02", "db-01"]  # 接続先を制限
            },
            "command": {
                "type": "string",
                "description": "Shell command to execute (read-only)"
            }
        },
        "required": ["host", "command"]
    }
}]
▶ enum でホストを制限  |  ▶ description に "read-only" と明示してモデルを誘導  |  ▶ 実装側 でもコマンド allowlist を別途かける（次スライド）

コード例 — エージェント呼び出し

import anthropic, subprocess

client = anthropic.Anthropic()
messages = [{"role": "user", "content": "web-01 でエラー急増アラート。調査して"}]

while True:
    response = client.messages.create(
        model="claude-opus-4-7",
        system="あなたはオンコール初動担当。error.log の急増を調査し、原因と推奨アクションを報告する。",
        tools=tools,
        messages=messages,
        max_tokens=4096,
    )
    if response.stop_reason == "end_turn":
        print(response.content[0].text)  # 調査レポートを Slack へ
        break
    # tool_use を実行して結果を返す
    for block in response.content:
        if block.type == "tool_use" and block.name == "run_ssh":
            stdout = run_ssh_safe(block.input["host"], block.input["command"])
            messages += [
                {"role": "assistant", "content": response.content},
                {"role": "user",      "content": [{"type": "tool_result",
                                                   "tool_use_id": block.id,
                                                   "content": stdout}]},
            ]
SDK が担う部分： tool_use_id の紐付け / context の詰め替え / トークン管理 / エラーリカバリー — 自前で書くと200行超える

ローテクだからこそ簡単

モダン環境（Datadog / k8s / OTel）
専用クライアント・認証・スキーマ理解が必要
MCP サーバーを自前で書く
API 変更に追従が必要
実装コスト：数日〜
ローテク環境（ファイルログ / SSH）
run_ssh を1つ書くだけ
LLM が一番得意な入力形式（プレーンテキスト）
SSH は成熟・冪等・監査ログが残る
実装コスト：数時間

枯れた道具ほど、エージェントと相性がいい

現実の落とし穴と次の一歩

  • ⚠️
    コマンド allowlist 必須
    rm / kill / shutdown / dd は実装側で弾く。description だけに頼らない
  • ⚠️
    出力サイズ制限
    run_ssh 側で | head -200 を強制注入。大きなログをそのまま返すとトークン爆発
  • ⚠️
    タイムアウト設定
    SSH コマンドに timeout 30s を付ける。ハングしたまま API コストが積まれる
  • 📋
    監査ログは別チャネルで保存
    Claude が実行したコマンドと stdout を DB に記録。振り返りと信頼構築に必須
